# db/ — DuckDB Setup

## What lives here
`setup.py` — the single script that initializes `powder_oracle.duckdb` from parquet files.

## When to run
Only when `powder_oracle.duckdb` is missing (gitignored). Do **not** run during a coding session — it takes a few minutes and is destructive (drops and recreates all tables).

```bash
PYTHONPATH=/Users/chli4608/Repositories/colorado_powder_oracle \
  /opt/anaconda3/envs/powracle/bin/python db/setup.py
```

## What setup.py does

### Phase 1 — Snowpack
1. Reads all CSVs from `data/raw/snotel/`, handles `.alias` pointer files
2. Computes `new_snow_24h`, `new_snow_48h`, `new_snow_72h` as day-over-day deltas (clipped at 0)
3. Adds `month` and `year` columns
4. Saves combined dataframe to `data/processed/snowpack.parquet`
5. Creates DuckDB table `snowpack` from the parquet
6. Creates views (see below)

### Phase 2 — Traffic
1. Reads `data/raw/traffic/traffic_history.csv` (generated by `ingestion/cdot_historical.py`)
2. Saves to `data/processed/traffic.parquet`
3. Creates DuckDB table `traffic_history`
4. Creates views (see below)

## Tables
| Table | Key columns |
|-------|-------------|
| `snowpack` | resort, date, snow_depth_in, swe_in, new_snow_24h, new_snow_48h, new_snow_72h, month, year |
| `traffic_history` | sensor_id, corridor, direction, timestamp, hour, day_of_week, month, year, is_holiday, volume |

## Views (what the tools actually query)
| View | Purpose | Key columns |
|------|---------|-------------|
| `monthly_averages` | Best month to ski a resort | resort, month, avg_depth_in, avg_swe_in, avg_daily_new_snow |
| `season_summary` | Above/below average season comparison | resort, year, peak_depth_in, peak_swe_in, total_new_snow_in |
| `traffic_patterns` | Best departure time lookup | corridor, direction, day_of_week, hour, month, is_holiday, avg_volume, stddev_volume |

## Typical tool queries

**Consistency ranking** (`get_snowpack_history` with "consistent" keyword):
```sql
SELECT resort, AVG(peak_depth_in) as avg_peak, STDDEV(peak_depth_in) as consistency_stddev
FROM season_summary GROUP BY resort ORDER BY consistency_stddev ASC
```

**Above/below average season** (current year vs 10yr avg):
```sql
SELECT resort, year, peak_depth_in,
       AVG(peak_depth_in) OVER (PARTITION BY resort) as historical_avg
FROM season_summary WHERE resort = ?
```

**Best departure time** (`get_best_departure_time`):
```sql
SELECT hour, avg_volume FROM traffic_patterns
WHERE corridor = ? AND direction = ? AND day_of_week = ? AND month = ?
ORDER BY avg_volume ASC LIMIT 3
```

## Adding new data to DuckDB
1. Add ingestion script to `ingestion/`
2. Add a new phase block in `db/setup.py` following the existing pattern (load → parquet → table → views)
3. Update the tables/views tables in this file
