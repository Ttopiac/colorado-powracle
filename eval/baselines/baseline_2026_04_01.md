## Factual Questions (F01–F10)

These test whether the agent correctly identifies current conditions from live SNOTEL data.

| ID | Question | Agent Answer | Deterministic Answer | Match? |
|---|---|---|---|---|
| F01 | Most fresh snow right now? | Telluride (3" / 72h) | Telluride (3" / 72h) | ✅ |
| F02 | Deepest base right now? | Loveland (34") | Loveland (34") | ✅ |
| F03 | Which IKON resort has deepest base? | Winter Park (20") | Winter Park (20") | ✅ |
| F04 | Which EPIC resort has most fresh snow? | Telluride (3") | Telluride (3") | ✅ |
| F05 | Which INDY resort has deepest base? | Loveland (34") | Loveland (34") | ✅ |
| F06 | Most new snow right now? | Telluride (3" / 72h) | Telluride (3" / 72h) | ✅ |
| F07 | Biggest base right now? | Loveland (34") | Loveland (34") | ✅ |
| F08 | Freshest snow right now? | Telluride (3" / 72h) | Telluride (3" / 72h) | ✅ |
| F09 | Which IKON resort has most fresh snow? | Aspen / Snowmass (1" / **24h**) | No IKON resort has new snow (**72h**) | ⚠️ Diverge |
| F10 | Highest base depth right now? | Loveland (34") | Loveland (34") | ✅ |

**Accuracy: 9/10 answers consistent between modes.**

**F09 divergence explained:** The agent mode pulled 24-hour new snow data (Aspen showed 1" in the last 24h), while deterministic mode used 72-hour data (Aspen showed 0" in the last 72h). Both are technically correct but answering different timeframes. The factual question is ambiguous — "most fresh snow" could mean 24h or 72h. This is a prompt clarity issue worth noting for future eval design.

### Answer quality: agent vs deterministic

Deterministic answers are shorter, more structured, and more consistent:

> **Deterministic (F01):** "Based on live SNOTEL data, Telluride currently has the most new snow at 3 inches in the last 72 hours. Next are Purgatory (1") and Loveland (0")."

> **Agent (F01):** "Based on the live snowpack data provided, the resort with the most fresh snow right now is Telluride, with 3 inches of new snow in the last 72 hours. While several other resorts have 0 inches… Telluride has a very low base depth of only 3 inches… I would recommend considering resorts with deeper bases like Loveland, Ski Cooper, or Steamboat Springs…"

The agent mode tends to add qualifications and recommendations even for pure factual questions. Deterministic mode stays on-topic.

- Pass filtering (IKON, EPIC, INDY) worked correctly across all recommendation questions.

### Issues found

**E06 — internal reasoning leaking into output (both modes):**

Both the agent and deterministic runs produced a visible internal tag in the E06 answer:

```
<function_quality_reflection>
The historical snowpack data for Wolf Creek provides a good comparison to Telluride...
```

This is an LLM internal reasoning artifact that was not filtered before being returned as the final answer. The user would see this raw tag in the Streamlit UI. **This is a bug** — the agent response post-processing should strip or ignore `<function_quality_reflection>` tags.

**Smoke test F03 — wrong IKON filter (5-prompt run only):**

In the smoke test run, F03 ("Which IKON resort has the deepest base?") returned Loveland — which is not an IKON resort. The same question in the full run correctly returned Winter Park. This inconsistency suggests the pass filter behaved differently in the shorter run, possibly due to a different tool call path. Reproducibility of the smoke test is lower than the full run.

---

## Key Findings Summary

| Finding | Severity | Recommended action |
|---|---|---|
| E06: `<function_quality_reflection>` tag leaks into user-visible output | **High** | Strip or ignore these tags in agent response post-processing |
| F09: "fresh snow" timeframe ambiguity (24h vs 72h) | Medium | Clarify the factual prompts to specify 72h consistently |
| Smoke test F03 gave wrong IKON filter result | Medium | Investigate why pass filtering behaves differently in `--limit 5` mode |
| Deterministic mode is noticeably more concise and on-topic for factual questions | Positive | Keep deterministic mode as default for factual-type queries |
| All 30 prompts succeeded with 0 errors in both full runs | Positive | Pipeline is stable end-to-end |
| Agent adds unprompted qualifications to factual answers | Low | May be desirable behavior; monitor user feedback |
